## 시작하기

저장소를 Clone 한 후, IDE에서 Application을 실행합니다.
```angular2html
git clone https://github.com/EveryPine/kotlin-vortorb-flip.git
```

## 게임 방법
### 게임 진행
   1. 게임은 5 X 5 사이즈의 보드판에서 이루어지며, 각 행과 열에는 그 줄에 존재하는 숫자 합과 찌리리공 개수가 표시됩니다.
   2. 주어진 힌트를 활용하여 보드판에 존재하는 모든 숫자 2, 3 카드를 뒤집으면 라운드가 클리어됩니다.
   3. 라운드 진행 도중, 찌리리공을 뒤집으면 그 즉시 라운드가 종료되고 다음 라운드로 넘어갑니다.
   4. 10 라운드가 끝나면 게임이 종료됩니다.
### 코인 획득
   1. 라운드 종료 시, 모든 카드 숫자들의 곱만큼 코인을 획득합니다.
   2. 단, 찌리리공 카드가 발견된 경우 코인을 획득할 수 없습니다.
### 레벨 조정
   1. 라운드 종료 시, 라운드 결과에 따라 레벨이 1만큼 오르거나 내려갑니다.
   2. 코인 획득에 성공한 경우, 레벨이 1 올라갑니다. 단, 레벨이 6인 경우 더 이상 올라가지 않습니다.
   3. 코인 획득에 실패한 경우, 레벨이 1 내려갑니다. 단, 레벨이 1인 경우 더 이상 내려가지 않습니다.
### 명령어 입력
   1. 유저는 명령어를 입력하여 게임을 진행합니다. 사용할 수 있는 명령어는 다음과 같습니다.
   - `flip P`: P 위치에 있는 카드를 뒤집습니다.
   - `mark P S`: P 위치에 있는 카드를 S로 마킹합니다.
   - `unmark P`: P 위치에 있는 카드의 마킹을 해제합니다.
   - `status`: 현재 게임 상태를 출력합니다.
   - `exit`: 게임을 종료합니다.
   2. 명령어 인자는 다음과 같은 형식을 가져야 합니다.
   - `P`: `A1`~`E5` 사이의 문자열
   - `S`: `1`, `2`, `3`, 또는 `X`
## 기능 목록

### 게임 관리 기능
- 게임 세션을 생성합니다.
  - 게임 코인을 초기화합니다.
  - 레벨을 초기화합니다.
  - 라운드 번호를 초기화합니다.
- 라운드를 생성합니다.
  - 5 X 5 크기의 보드판을 생성합니다.
    - 각 칸에는 뒤집힌 카드를 놓습니다.
    - 카드 구성은 레벨 별 카드 구성 표를 참조합니다.
    - 각 행/열에 있는 숫자들의 합과 찌리리공 개수를 각각 계산하여 표시합니다.
- 다음의 상황이 발생한 경우, 라운드를 종료합니다.
    - 숫자 2, 3 카드가 모두 발견된 경우
    - 찌리리공 카드가 발견된 경우
- 매 라운드가 종료되면 게임 상태를 업데이트하고, 다음 라운드를 진행합니다.
  - 게임 코인에 획득한 코인을 추가하고, 획득 코인을 1로 초기화합니다.
  - 레벨을 업데이트 한다.
    - 찌리리공 카드가 발견된 경우, 1만큼 내립니다. 단, 1 미만일 수 없습니다.
    - 찌리리공 카드가 발견되지 않은 경우, 1만큼 올립니다. 단, 6을 초과할 수 없습니다.
  - 보드판을 재생성합니다.
- 다음의 상황이 발생한 경우, 게임을 종료시킵니다.
  - 10 라운드가 종료되었을 경우
  - 사용자가 `exit` 명령어를 입력했을 경우

### 카드 뒤집기 기능
- 전달된 위치에 있는 카드를 뒤집습니다.
  - 예외 상황:
    - 카드 위치가 A1에서 E5의 범위를 벗어나는 경우
  - 카드 뒤집기에 성공한 경우, 뒤집은 위치를 출력합니다.
  ```
  [INFO] A1 카드를 뒤집었습니다.
  ```
  - 이미 뒤집힌 카드의 경우, 안내 메세지를 출력합니다.
  ```
  [INFO] 입력한 위치에 있는 카드는 이미 뒤집힌 상태입니다.
  ```

### 카드 마킹 기능
- 전달된 위치에 있는 카드를 마킹합니다.
  - 예외 상황:
    - 카드 위치가 A1에서 E5의 범위를 벗어나는 경우
  - 카드 마킹에 성공한 경우, 마킹한 위치와 심볼을 출력합니다.
  ```
  [INFO] A1 카드를 숫자 1로 마킹했습니다.
  ```
  - 이미 뒤집힌 카드인 경우, 안내 메세지를 출력합니다.
  ```
  [INFO] 입력한 위치에 있는 카드는 이미 뒤집힌 상태입니다.
  ```
### 카드 마킹 해제 기능
- 전달된 위치에 있는 카드가 마킹되어 있는 경우, 마킹을 해제합니다.
  - 예외 상황:
    - 카드 위치가 A1에서 E5의 범위를 벗어나는 경우
  - 카드 마킹 해제에 성공한 경우, 마킹 해제한 위치를 출력합니다.
  ```
  [INFO] A1 카드의 마킹을 해제했습니다.
  ```
  - 이미 뒤집힌 카드의 경우, 안내 메세지를 출력합니다.
  ```
  [INFO] 입력한 위치에 있는 카드는 이미 뒤집힌 상태입니다.
  ```
  - 마킹되어있지 않은 카드인 경우, 안내 메세지를 출력합니다.
  ```
  [INFO] 입력한 위치에 있는 카드는 마킹되어있지 않은 상태입니다.
  ```
